# ARCHIVE CANDIDATES

이 파일은 유지보수 관점에서 보관(archival) 또는 정리(candidate) 대상으로 제안하는 문서 목록과 간단한 이유를 정리합니다. 실제 이동/삭제는 사용자 확인 후 진행합니다.

제안 보관 대상:

- `docs/err/` 아래의 모든 `Task*_err.md` 파일 — 오류/임시 로그 성격의 문서로, 문서화된 사양이 아니며 참조 빈도가 낮음.

보관 절차 제안:
1. 이 목록을 확인 후 승인해주세요.
2. 승인 시 `docs/archived/err/`로 이동(원본 삭제, 복사본 보관).
3. 필요 시 Git 커밋 메시지에 보관 사유 표기.

참고: `PredictionVerification-TestPlan.md` 등 일부 파일은 내부 레퍼런스(`.omc/notepad.md` 등)에 존재하므로 보관하지 않았습니다.
